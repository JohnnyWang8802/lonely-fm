打开频道，连接自己的 Gemma 4。

测试入口

先从线上前端进入。登录后页面会检测你电脑上的本地后端和 Ollama / Gemma 4。

打开 Lonely FM

测试前准备

这次不是单纯打开网页就结束。Vercel 只负责前端页面，Gemma 4 需要用你自己电脑上的 Ollama 和本地后端来跑。

安装并启动 Ollama

如果还没有 Ollama，先安装：https://ollama.com/download

ollama serve

确认本地有 Gemma 4

Lonely FM 兼容模型名以 gemma4 开头的版本，例如 gemma4:12b-mlx、gemma4:e4b、gemma4:21b。

ollama list

如果还没有 Gemma 4，可以先拉一个版本：

ollama pull gemma4:12b-mlx

允许 Vercel 页面访问本地 Ollama

如果你在 Mac 上测试，先执行下面这句，然后退出并重新打开 Ollama。

launchctl setenv OLLAMA_ORIGINS "https://lonely-fm.vercel.app,http://localhost:5173,http://127.0.0.1:5173"

启动 Lonely FM 本地后端

如果已经有项目代码，进入项目目录后执行：

cd backend
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn main:app --host 127.0.0.1 --port 8001

后端启动后，打开这个地址确认是否有响应：

http://127.0.0.1:8001/health
如果没有项目代码，先从 GitHub 拉取 Lonely FM 仓库，或让 Johnny 把后端代码发给你。

登录和进入频道

登录用于云端保存你愿意留下的记忆。访客模式可以临时体验，但离开后不会保存本轮记忆。

打开线上页面

https://lonely-fm.vercel.app/

点击“进入频道”，输入邮箱，然后点击“获取登录邮件”。

完成邮箱登录

打开收到的登录邮件，直接点击邮件里的确认链接完成登录。

如果 1-2 分钟没收到，检查垃圾邮件。当前临时使用 Supabase 内置邮件服务，免费额度很低，短时间多次测试可能需要等一会儿。

检测 Gemma 4 并选择声音

登录后会进入 Gemma 连接页。检测通过后选择林屿或阿婉，进入连接中页面。接通后，角色会先主动打招呼。

重点测什么

这次测试重点不是看页面是否能打开，而是看“语音陪伴”是否真的自然、稳定、能接住用户的话。

语音链路

能不能听到回复，回复前有没有很长空白，声音是否卡顿、重叠或中断。

理解质量

林屿/阿婉是否真的针对你刚说的话回应，而不是说套路话、答非所问。

记忆体验

登录后多轮对话，是否能保留和使用用户愿意留下的信息。
